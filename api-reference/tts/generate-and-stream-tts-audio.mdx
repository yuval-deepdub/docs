## Supported output formats

The REST API returns a complete audio file as raw bytes in the response body.

| Format | Content type | Description |
|--------|-------------|-------------|
| `mp3` | `audio/mpeg` | Compressed audio, smallest file size. **Default.** |
| `wav` | `audio/wav` | Uncompressed PCM audio with WAV header. |
| `opus` | `audio/opus` | High-quality compressed audio, efficient for streaming. |
| `mulaw` | `audio/basic` | 8-bit µ-law encoding, commonly used in telephony. |

## Supported sample rates

| Sample rate | Description |
|-------------|-------------|
| `8000` | Telephony (µ-law) |
| `16000` | Wideband speech |
| `22050` | Standard quality |
| `24000` | Enhanced speech |
| `32000` | High quality |
| `36000` | Studio speech |
| `44100` | CD quality |
| `48000` | Professional audio |

<Note>
  Not all format and sample rate combinations are supported. For `mulaw`, use `8000` Hz. For `mp3`, sample rates above `48000` Hz are not supported.
</Note>

### REST vs WebSocket differences

| Feature | REST API | WebSocket API |
|---------|----------|---------------|
| **Delivery** | Complete audio file in a single response | Chunked audio delivered incrementally as base64-encoded messages |
| **Formats** | `mp3`, `wav`, `opus`, `mulaw` | `mp3`, `wav` (headerless), `opus`, `mulaw` |
| **WAV behavior** | Returns a complete WAV file with headers | Returns raw PCM audio bytes without WAV headers |
| **Sample rates** | 8000, 16000, 22050, 24000, 32000, 36000, 44100, 48000 | 8000, 16000, 22050, 24000, 32000, 36000, 44100, 48000 |
| **Best for** | Simple integrations, file generation | Real-time playback, low-latency applications |
